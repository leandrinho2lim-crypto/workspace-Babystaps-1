# Minhas Anotações de Aula 📚

## 🌿 Versionamento com Git (Prof. Gabriel)

### 📖 Teoria e Conceitos
* **História do Git:** Criado para gerir o desenvolvimento do Linux, focado em velocidade e integridade de dados.
* **Para que serve:** Controlar as diferentes versões de um ficheiro ao longo do tempo.
* **Vantagens do Git:**
    * Permite trabalhar em equipa sem sobrescrever o código dos outros.
    * Histórico completo de todas as alterações feitas.
    * Facilidade em testar novas ideias através de branches.

### 🛠️ Tarefas Práticas Realizadas
* **Criação de conta:** Configuração do perfil no GitHub.
* **Segurança (SSH):** 
    * Geração da chave SSH no Git Bash para autenticação sem password.
    * Associação da chave SSH pública no perfil do GitHub.
* **Fluxo Inicial:**
    * **Criação de Repositório:** Setup inicial diretamente no GitHub.
    * **Clone:** Uso do comando `git clone` para trazer o projeto da nuvem para a minha máquina.

## Git & GitHub

### 🌿 Organização e Comandos
* **Markdown (.md):** Linguagem de marcação para criar documentos formatados (textos, negritos, links).
* **Gestão de Repositórios:** Como criar e excluir repositórios no GitHub.
* **Branches:** Criação de "ramos" para trabalhar em partes diferentes do projeto sem afetar o principal.

### 🛠️ Comandos de Limpeza (Pesquisa)
* `git branch -d nome`: Apagar uma branch localmente.
* `git push origin --delete nome`: Apagar uma branch no servidor.
